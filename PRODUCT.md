# Typori — Product Context

## register
brand

## Product Purpose
Typori is a font-testing webapp for typesetters, graphic designers, and type designers who need to see a typeface working in realistic contexts before committing to it. The tool lets you load any local font, input text in any script, and see it rendered inside preset web and print layouts. It is script-agnostic and UTF-8 native — designed explicitly to support non-Latin scripts like Odia, Devanagari, Arabic, and others.

## Users
- **Type designers** testing a typeface they are still drawing, checking how it renders in realistic contexts at multiple sizes
- **Graphic designers** evaluating a licensed or custom font for a specific project (magazine, app, packaging)
- **Typesetters** proofing editorial text in candidate typefaces before layout

All users are experts. The tool should not explain typography to them. It should get out of the way and let the font speak.

## Brand Personality
Rational, authoritative, utilitarian. The UI recedes. The type takes center stage. No decoration that doesn't serve a function. Inspired by Swiss International Typographic Style: rigorous grid, flush-left, strong vertical rhythm. The tool is a precision instrument, not a consumer app.

## Anti-references
- Not Canva (approachable, bubbly, template-driven)
- Not Adobe Fonts (marketplace, discovery-oriented)
- Not Google Fonts (specimen browser, not a testing tool)
- Not Type-is-beautiful blog (editorial, read-only)

## Strategic Principles
1. Script-agnostic first: the tool works identically regardless of script direction, shaping, or glyph complexity
2. Presets over configuration: 5–6 realistic layout contexts per mode, not an infinite configuration surface
3. Load → See → Export: the entire flow in three steps
4. GitHub Pages static: no server, no auth, no accounts
